Use esta estrutura como base. Omitir seções vazias e adaptar a ordem apenas quando isso melhorar a leitura.

```markdown
## Resumo

<descrever o objetivo principal da PR e o motivo da mudança>

## O que mudou

- <mudança concreta 1>
- <mudança concreta 2>

## Validação

- [ ] testes automatizados executados
- [ ] validação manual realizada
- [ ] documentação atualizada

## Riscos e observações

- <riscos, limitações, rollout, feature flag ou contexto de revisão>

## Breaking changes

- <preencher apenas quando houver quebra confirmada>

## Ticket ou contexto adicional

- <jira, incidente, contexto operacional ou decisão arquitetural>
```

Resumo operacional para retorno ao usuário:

```text
PR: <rascunho|criada|atualizada>
URL: <url-ou-n/a>
Base: <base-branch>
Título: <titulo-final>
```
